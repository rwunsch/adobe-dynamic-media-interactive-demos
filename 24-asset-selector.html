Dynamic Media OpenAPI · Embeddable UI

Pick an asset, get a delivery URL

The Adobe Asset Selector is an embeddable micro-frontend you drop into any app — a CMS, an email tool, a storefront. The user browses governed AEM assets in a familiar UI and picks one; the selector hands your app back the asset's stable Dynamic Media OpenAPI delivery URL. No DAM login in your app, no asset copies, no path guessing — sign in once with Adobe IMS and the picker returns a URL you can paste straight into an <img>.

Why this matters

The selector is the same component AEM uses internally — provisioned to your IMS org, it enforces permissions and governance. Your app never holds AEM credentials; it only receives a delivery URL for an asset the user is allowed to see. That URL (/adobe/assets/urn:aaid:aem:<uuid>/as/<name>.jpg) is the stable, approval-gated OpenAPI endpoint — drop it into any HTML, email or product page.

How to get a client ID

The in-browser sign-in needs an Asset-Selector provisioned imsClientId. Adobe Developer Console client IDs do not work, and Adobe ships no public sample — every official example uses a <YOUR_IMS_CLIENT_ID> placeholder. Two ways to get a real one:

A · Provision one (official):

Your program admin raises a P2 support ticket from the Admin Console requesting Asset Selector / Content Advisor provisioning, including the Program ID + Environment ID of the AEM CS instance and the domain name(s) where this app is hosted (the allowed origins). Adobe returns an imsClientId, imsScope and redirectUrl per environment.

B · Reuse an existing one (if your org already runs it):

If your org already uses Content Hub or the AEM Assets view at experience.adobe.com, open it signed-in, open browser DevTools → Network, filter for ims/authorize, and read the client_id on the IMS auth request. That same provisioned id can drive this selector — paste it into CONFIG.imsClientId. (Quickest interim path: just use the paste-a-token fallback above — no client id needed.)
