Dynamic Media · Templating & personalization

One template, a different banner per segment

The designer builds one layered template with named visibility and text variables. Marketing then defines a small bundle of parameters per audience segment — which layers show, what the headline says. The same template URL renders a tailored creative for each segment, driven entirely by data. No new files, no Photoshop round-trip. This runs live on the real tenant template DMS7-Bunny-S7.

Audience segment

Each button is a saved segment recipe — a bundle of parameter values. In production these come from your campaign feed or CDP, one row per segment.

Headline for this segment

The headline is just another segment variable ($cr-text). Edit it to see this segment update live.

Why segment templating

The design (layout, fonts, the layer stack, positions) is authored once and frozen in the template. The data (which scene, which headline) is a tiny per-segment payload. Swap the payload → new creative. Scale to thousands of localized / seasonal / audience variants from a single source of truth.
