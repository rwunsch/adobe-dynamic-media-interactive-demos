Dynamic Media OpenAPI · Headless & data

One API for the pixels and the data

The same Dynamic Media OpenAPI that delivers the rendered image also returns the asset's metadata as JSON — title, dimensions, format, size, review status — straight from AEM. One governed endpoint, two payloads. No scraping, no separate DAM lookup, no second system. Pick an asset and watch your browser fetch both the rendition and its live metadata.

Why this matters

A front-end calls one Dynamic Media OpenAPI host. The /as/… path streams the optimized image; the /metadata path returns the governed field data as JSON. The response is CORS-open (access-control-allow-origin: *), so an in-browser app can read it directly.

This is the headless story: it powers Content Hub, dynamic galleries, product detail pages and app integrations — pixels and field data from a single, approval-gated source of truth.
